Content Standard

The Content Standard is a set of 5 principles to help content teams create and manage good content.

This is dummy text. New text needs to be created.

1. Content must be needed

All content must be designed to meet user needs before government needs. If there isn't a user need for your content, then you shouldn't publish it. User needs should be supported by evidence.

Read more about meeting user needs

2. Content must be effective

Every piece of content has a job to do. Its purpose should be clear to the user from its title and description. It must be well structured and use clean language to meet user needs.

Read more about effective content

3. Content must be well organised

Content must be published in the appropriate format and be tagged correctly so it appears where users expect to find it.

Read more about organising content

4. Content must be accessible

Government content must be for everyone and must meet accessibility requirements. It should be usable wherever and however it might be viewed, for example on small devices and when republished elsewhere.

Read more about accessible content

5. Content must be actively managed

Cotnent must have an owner responsible for maintaining it throughout its lifecycle. There must be a process for regular content review to ensure content is stil needed and accurate, withdrawn or unpublished as appropriate. The need for a content review will vary according to content type.

Read more about managing content
